\subsection{Typen in Python}

\begin{frame}[fragile]{Dynamische Typisierung}
  In Python haben Variablen einen Namen und einen Wert, aber keine benennbare Speicheradresse. 
  Variablen sind \tb{dynamisch typisiert}. 
  Ihr Typ\footnote{
    Der Typ einer Variable oder einer Expression \texttt{EXPR} wird mithilfe von \texttt{type(EXPR)} zurückgegeben.
  } ist zur Laufzeit änderbar.
  \begin{lstlisting}[language=pytiny]
meine_variable = 1              # >> type(meine_variable): <int>
meine_variable = "never gonna"  # >> type(meine_variable): <str>
meine_variable = KlasseA()      # >> type(meine_variable): <KlasseA>
  \end{lstlisting}

  Um den Programmcode nachvollziehbar zu gestalten, ist eine klare Namensgebung und Einhaltung von Typen einer Variable sehr wünschenwert. 

  \medskip

  Um den \tb{vorgesehenen}, nicht erzwungenen Typ einer Variable für Entwickler:innen und IDEs\footnote{
    Siehe z.B.\ \url{https://www.jetbrains.com/help/pycharm/type-hinting-in-product.html} .
  } zu dokumentieren, verwendet man \tb{Typehinting}\footnote{\url{https://docs.python.org/3/library/typing.html}.
  }.
  
\end{frame}

\begin{frame}[fragile]{Ungeordnete Menge}
  Der Typ \pp{set} realisiert eine \tb{ungeordnete Menge}. Ein \pp{set} implementiert Funktionalität für \tb{Elementtest}, und die grundlegenden \tb{mathematischen} Operationen auf Mengen.\footnote{

  \begin{lstlisting}[language=pytiny]
basket = {'apple', 'orange', 'apple', 'pear', 'banana'}  # >> {'orange', 'banana', 'pear', 'apple'}
'orange' in basket     # >> True
'crabgrass' in basket  # >> False

a = set('abracadabra')  # >> {'a', 'r', 'b', 'c', 'd'}
b = set('alacazam')     # >> {'a', 'l', 'c', 'z', 'm'}
    
d = a - b  # >> {'r', 'd', 'b'} Differenz

d.add(42)                     # >> {'r', 'd', 'b', 42}
d.intersect_update({'r', 3})  # >> {'r'}
  \end{lstlisting}

Equivalent zu \pp{-} gibt es \pp{|} für die Vereinigung, \pp{&} für den Schnitt und \pp{^} für die symmetrische Differenz.

        \item[] \begin{itemize}

\subsection{Externe Bibliotheken}

\begin{frame}[fragile]{Module einbinden}
  In Python gibt es zwei Möglichkeiten, fremde Module (externe Bibliotheken) einzubinden. 
  \begin{lstlisting}[language=pytiny]
import numpy as np  # Importiert das gemsate Modul 'numpy' und bennent es in 'np' um.
array = np.array()  # Zugriff über Modulname.Funktionalität

from random import randrange  # Die Funktion 'randrange' wird in den eigenen Namespace importiert
print(f"Dieser Code ist eine {1+randrange(10)} von 10")
  \end{lstlisting}

  Das Einbinden von fremder Funktionalität via \lpl{from module import *} 
  widerspricht dem Konzept der Namespaceunterteilung und soll nicht verwendet werden.

  \nsl{

\end{frame}

\begin{frame}[fragile]{Module beziehen}
  Üblicherweise werden Pythonmodule auf dem \tb{Python Package Index}, kurz \tb{PyPI}, veröffentlicht. 
  Das Veröffentlichen von Modulen auf PyPI ist vergleichsweise einfach,
  denn der Modulcode wird nicht inhaltlich geprüft. 
  Das ist auch für bösartige Softwareentwickler:innen sehr praktisch. 

  \medskip

  Um ein ausgewähltes Modul von PyPI zu beziehen, verwendet man \lpl{pip}. 
  \begin{lstlisting}[language=pltiny]
python3 -m pip install --user MODULNAME
  \end{lstlisting}

  Optional erzeugt man zuerst ein zugehöriges \tb{Virtual Environment}\furl{https://packaging.python.org/en/latest/guides/installing-using-pip-and-virtual-environments/}
  und bezieht darin die gewünschten Module mithilfe von \lpl{pip}.
