\subsection{Typen in Python}

\begin{frame}[fragile]{Dynamische Typisierung}
  In Python haben Variablen einen Namen und einen Wert, aber keine benennbare Speicheradresse. 
  Variablen sind \tb{dynamisch typisiert}. 
  Ihr Typ\footnote{
    Der Typ einer Variable oder einer Expression \texttt{EXPR} wird mithilfe von \texttt{type(EXPR)} zurückgegeben.
  } ist zur Laufzeit änderbar.
  \begin{lstlisting}[language=pytiny]
my_variable = 1              # >> type(my_variable): <int>
my_variable = "never gonna"  # >> type(my_variable): <str>
my_variable = ClassA()       # >> type(my_variable): <KlasseA>
  \end{lstlisting}

  Um den Programmcode nachvollziehbar zu gestalten, ist eine klare Namensgebung und Einhaltung von Typen einer Variable sehr wünschenwert. 

  \medskip

  Um den \tb{vorgesehenen}, nicht erzwungenen Typ einer Variable für Entwickler:innen und IDEs\footnote{
    Siehe z.B.\ \url{https://www.jetbrains.com/help/pycharm/type-hinting-in-product.html} .
  } zu dokumentieren, verwendet man \tb{Typehinting}\footnote{\url{https://docs.python.org/3/library/typing.html}.
  }.
  
\end{frame}


% Hier erwähnen das Typhinting nicht bindend ist.

\begin{frame}[fragile]{Ungeordnete Menge}
  Der Typ \pp{set} realisiert eine \tb{ungeordnete Menge}. Ein \pp{set} implementiert Funktionalität für \tb{Elementtest}, und die grundlegenden \tb{mathematischen} Operationen auf Mengen.\footnote{

  \begin{lstlisting}[language=pytiny]
basket = {'apple', 'orange', 'apple', 'pear', 'banana'}  # >> {'orange', 'banana', 'pear', 'apple'}
'orange' in basket     # >> True
'crabgrass' in basket  # >> False

a = {'abracadabra'}    # >> {'a', 'r', 'b', 'c', 'd'}
b = set('alacazam')    # >> {'a', 'l', 'c', 'z', 'm'}
    
d = a - b              # >> {'r', 'd', 'b'} Differenz

d.add(42)                     # >> {'r', 'd', 'b', 42}
d.intersect_update({'r', 3})  # >> {'r'}
  \end{lstlisting}

	Equivalent zu \pp{-} gibt es \pp{|} für die Vereinigung, \pp{&} für den Schnitt und \pp{^} für die symmetrische Differenz.

\begin{frame}[fragile]{Vorsicht bei der Erstellung von Tuplen}
   Entscheidend für die \tb{Konstruktion} des Tuples ist das \tb{Komma}.
   
   \medskip
   \pause
   
   Dadurch kann es schnell zu Verwirrung bei der Deklaration kommen.\newline
   Insbesondere können Ausdücke, die offensichtlich wie ein Tuple aussehen, keins sein.
   \pause
   \begin{lstlisting}[language=pytiny]
a = (2, 3)     # >> (2, 3)
b = 2, 3       # >> (2, 3)
c = (2,)       # >> (2,)
d = 2,         # >> (2,)
e = (2)        # >> 2 (int, not tuple of int!)
f = (1 == 1)   # >> True (also not a tuple, just bool)
g = (1 == 1,)  # >> (True,)
   \end{lstlisting}

	Es ist best-practice die Konstruktion eines Tuples \tb{immer} zu \tb{klammern}, außer bei der Rückgabe mehrerer Werte am Ende einer Funktion.
   
\end{frame}

\subsection{Ein einheitlicher Coding Style}
\begin{frame}{Warum einen Coding Style nutzen?}
        \begin{itemize}
	\item Viele Programmiersprachen lassen viel \tb{Freiraum} für das Formatieren des Codes.
	\item Hierduch kann das selbe Programm sehr \tb{unterschiedlich} aussehen.
	\pause
	\item Damit sich Veränderungen vieler Personen an der selben Codebase \tb{einfacher mergen} lassen, werden \tb{Coding Styles} definiert.
	\pause
	\item Außerdem helfen sie sich schneller \tb{in Projekten orientieren} zu können und eine gemeinsame, \tb{einheitliche Codebase} zu erschaffen.
	\pause
	\item Im Allgemeinen werden solche Styles von der Community als \tb{Konvention} angenommen, wobei große Projekte in der Regel einen Style erzwingen.
	\item Für die meisten Coding Styles existieren \tb{Code Formatter}, welche den Code automatisch in die gewünschte Form bringen.
	\item Ein sehr bekannter Style-Guide ist der für den Linux Kernel\footnote{\url{https://www.kernel.org/doc/html/v4.10/process/coding-style}}.
\end{itemize}

  In Python gibt es zwei Möglichkeiten, fremde Module (externe Bibliotheken) einzubinden.
