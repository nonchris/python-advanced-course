\section{Discord Bots}
\subsection{Async in einer Folie}

\begin{frame}{Async: Informell in vier Punkten + Fazit}
	\begin{itemize}
		\item \tb{Asynchrone} Programme können zwischen der \tb{Ausführung} verschiedener Funktionen \tb{wechseln}, wenn eine Funktion \tb{blockierend} auf (i.d.R.) I/O wartet.
		
		\pause
		
		\item[] Hierbei läuft das gesamte Programm immernoch in \tb{einem} Thread\footnote{Dennoch ist die Ausführung des Programms unvorhersehbar, daher kann es notwendig sein, Ressourcen zu schützen!}.
		
		\pause
		
		\item Ist eine \tb{Funktion} als \pp{async} gekennzeichnet, bedeutet dies, dass sie irgendwo auf (i.d.R.) I/O \tb{warten} wird.
		
		\pause
		
		
		\item Solche Methoden nennen sich \tb{Coroutinen}.
		
		\pause
		
		\item Coroutinen \tb{müssen}, im Kontext von Discord.py, direkt mit \pp{await} \tb{abgewartet} werden.
		
		\pause
		
		\item[] \tb{Sobald} das Programm auf \pp{await} trifft, kann es an einer \tb{anderen} Stelle weiter arbeiten und zu diesem Aufruf \tb{zurückkehren}, sobald die Coroutine abgeschlossen ist.
		
		\item \tb{Fazit}: Durch \pp{asyncIO} wird \tb{keine Zeit} mit Warten auf I/O \tb{verschwendet}.
		
	\end{itemize}
\end{frame}

\subsection{Überblick}

\begin{frame}{Was kann ein Bot}
	Discord \tb{Bots} sind Programme, welche mit Discords API\furl{https://discord.com/developers/docs/reference} interagieren, um \tb{Aktionen} innerhalb der Plattform durchzuführen.
	
	\medskip
	
	In Discord sind Bots eine besondere \tb{Klasse} von Accounts\furl{https://discord.com/developers/docs/topics/oauth2\#bot-vs-user-accounts}, welche in der App durch das \pp{BOT}-Label \tb{gekennzeichnet} sind.
	
	\medskip
	
	Diese Accounts interagieren mit Discord über eine \tb{REST-API}.
	
	\medskip
	
	Damit ein Bot diese nutzen kann, benötigt er ein \tb{Token} für die API\furl{https://discord.com/developers/docs/reference\#authentication}.
	
\end{frame}

\begin{frame}{Eine Library für einfache Interaktion mit Discords API in Python: discord.py}
	Für gängige Webdienste existieren zumeist \tb{Libraries}, welches die Interaktion mit\newline
	deren API stark \tb{vereinfachen}.
	
	\medskip
	
	Das Ökosystem um Pythons Discord Libraries ist seit der temporären Einstellung der Entwicklung von \pp{discord.py} 2021\furl{https://gist.github.com/Rapptz/c4324f17a80c94776832430007ad40e6} 
	breit aufgestellt.
	
	\medskip
	
	Da \pp{discord.py}\furl{https://discordpy.readthedocs.io/en/stable/}
	die \tb{größte} Library für Interaktion mit Discord via Python ist und seit Anfang 2022 wieder aktiv maintained wird\furl{https://gist.github.com/Rapptz/4a2f62751b9600a31a0d3c78100287f1}
	, führen wir hier in diese ein\footnote{Gezeigt wird die im August 2022 veröffentlichte \texttt{v2}}.
	
\end{frame}

\subsection{discord.py - Überblick}

\begin{frame}{Die wichtigsten Objekte in Kürze}
	\begin{itemize}
		\item Das \pp{discord.Guild}\furl{https://discordpy.readthedocs.io/en/latest/api.html\#guild} modelliert einen Discord-Server.
		
		
		\item Das \pp{discord.Member}\furl{https://discordpy.readthedocs.io/en/latest/api.html\#member} modelliert ein Mitglied eines Servers.
		
		
		\item Das \pp{discord.TextChannel}\furl{https://discordpy.readthedocs.io/en/latest/api.html\#textchannel} modelliert einen Text-Kanal auf einem Server.
		
		
		\item Das \pp{discord.Message}\furl{https://discordpy.readthedocs.io/en/latest/api.html\#message} modelliert eine versendete Nachricht.
		
		\item Das \pp{discord.ext.commands.Bot}\furl{https://discordpy.readthedocs.io/en/latest/ext/commands/api.html\#bot} realisiert unseren \tb{Bot-User}.
		
		\item Das \pp{discord.ext.commands.Context}\footnote{\url{https://discordpy.readthedocs.io/en/latest/ext/commands/api.html\#context}, bei Slash-Commands ist es stattdessen das \texttt{Interaction}-Objekt.} enthält den Kontext eines Commands.
		
		\item Jedes Element hat eine \tb{eindeutige} von Discord zugewiesene \pp{id}.
		
		\item Im Allgemeinen sind alle einem Kontext zugehörige Objekte voneinander erreichbar.
	\end{itemize}
\end{frame}


\begin{frame}{Aufsetzen eines Bots - Die minimalen Anforderungen}
	\begin{itemize}
		\item Wir brauchen eine \tb{Instanz} von \pp{discord.ext.commands.Bot}\footnote{
			\texttt{Bot} ist eine Erweiterung von \texttt{discord.Client}: https://discordpy.readthedocs.io/en/latest/api.html\#discord.Client} oder einer davon erbenden Klasse.
		
		\pause
		
		\item Die \pp{Bot}-Klasse benötigt zur Instanziierung ein \tb{Prefix} und eine \tb{Intent}-Konfiguration\furl{https://discordpy.readthedocs.io/en/stable/api.html\#discord.Intents}.
		
		\pause
		
		\item Das \pp{Bot}-Objekt kann mit einem \pp{token} und der \pp{run()}-Methode \tb{gestartet} werden.
		
		\pause
		
		\item Es empfiehlt sich, direkt die \pp{setup_hook}\furl{https://discordpy.readthedocs.io/en/stable/ext/commands/api.html?highlight=setup_hook\#discord.ext.commands.Bot.setup_hook} mit zu \tb{überschreiben}.
		
		\pause
		
		\item Das \pp{Bot}-Objekt kennt \tb{alle} Server, Privatchats etc. des Bot-Clients. Es stellt eine Reihe von \tb{Gettern} zur Verfügung, um leicht auf diese Elemente zuzugreifen.
	\end{itemize}
\end{frame}

\begin{frame}[fragile]{Ein ziemlich minimaler Discord-Bot}
	\begin{lstlisting}[language=pytiny]
		import discord
		from discord.ext import commands
		
		class MyBot(commands.Bot):
		def __init__(self, intents: discord.Intents = discord.Intents.all()):
		super().__init__(command_prefix="b!", intents=intents)
		
		async def setup_hook():
		""" load your cogs here, also push the app commands"""
		await self.load_extension("my_sick_commands", package=__package__)
		
		self.tree.copy_global_to(guild=self.get_guild(int(os.getenv("MY_GUILD"))))
		await self.tree.sync(guild=guild)
		print(f"Bot is ready to go")
		
		if __name__ == "__main__":
		bot = MyBot()
		bot.run(os.getenv("TOKEN"))
	\end{lstlisting}
\end{frame}


\begin{frame}{Projekte gliedern mit Cogs}
	\begin{itemize}
		\item Schnell ist der Punkt erreicht, wo zusammengehörende \tb{Funktionalitäten} gekapselt in ein eigenes Modul \tb{ausgelagert} werden sollen.
		
		\pause
		
		\item Discord.py bietet dafür die \pp{commands.Cog}-Klasse\furl{https://discordpy.readthedocs.io/en/latest/ext/commands/api.html\#cog}.
		
		\pause
		
		\item \tb{Erbt} ein Objekt von dieser, kann es zur Laufzeit in den Bot \tb{geladen} werden.
		
		\pause
		
		\item Ein Modul, welches \pp{Cogs} enthält, \tb{benötigt} eine \pp{setup()}-Funktion.
		
		\pause
		
		\item Es wird via \pp{load_extension()}\furl{https://discordpy.readthedocs.io/en/latest/ext/commands/api.html\#discord.ext.commands.Bot.load_extension} in den Bot geladen.
		
		\pause
		
		\item Ein Cog existiert nur einmal und kann \tb{Command-übergreifend} Daten halten.
		
		\pause
		
		\item Cogs können im laufenden Betrieb neu geladen werden\furl{https://discordpy.readthedocs.io/en/latest/ext/commands/api.html\#discord.ext.commands.Bot.reload_extension}.
	\end{itemize}
\end{frame}


\begin{frame}[fragile]{Ein ziemlich kleiner Cog}
	\begin{lstlisting}[language=pytiny]
		# this could be the module 'my_sick_commands' mentioned in the previous example
		import discord
		from discord.ext import commands
		
		class UppercaseCog(commands.Cog):
		# cog takes the bot (and should save it internally) - we can store arbitrary data here!
		def __init__(self, bot):
		self.bot = bot
		
		# we need commands.command in Cogs and bot_instance.command in our main file
		@commands.command()
		async def uppercase(self, ctx: commands.Context, word: str):
		""" a command to convert a single word to upper case """ 
		await ctx.send(word.upper())
		
		# special load function, required to be able to load the module
		def setup(bot):
		bot.add_cog(UppercaseCog(bot))
	\end{lstlisting}
	
	\nll{Auf await hinweisen!}
\end{frame}

\subsection{Event-basierte Operationen}
\begin{frame}{Event-basierte Operationen: Überblick}
	\tb{Event}-basierte Operationen sind Operationen, die der Bot durchführt, wenn ein\newline
	bestimmtes Event \tb{aufgetreten} ist.
	
	\medskip
	\pause
	
	Bots können auf eine ganze Reihe von Events\furl{https://discordpy.readthedocs.io/en/latest/api.html\#event-reference} hören. Unter Anderem:
	%
	\begin{itemize}
		\item Empfang einer Nachricht \pp{on_message}
		\item Join eines Members \pp{on_member_join}
		\item Hinzufügen / Entfernen einer Reaction \pp{on_reaction_add} bzw. \pp{_remove}.
		\item Die Veränderung eines Members \pp{on_member_update}
		\item Oder auch einfach das Tippen in einem Channel \pp{on_typing}
	\end{itemize}
	
	\medskip
	\pause
	
	Events werden durch das \tb{Überschreiben} der Methode des Bots gehandelt.
	
	\medskip
	\pause
	
	Hinweis: Viele dieser Events erfordern \tb{Privileged} Gateway Intents\furl{https://discord.com/developers/docs/topics/gateway\#privileged-intents}\furl{https://support-dev.discord.com/hc/en-us/articles/6207308062871-What-are-Privileged-Intents-}.
	
\end{frame}


\begin{frame}[fragile]{Eventhandler überschreiben}
	\begin{lstlisting}[language=pytiny]
		import discord
		from discord.ext import commands
		
		class MyBot(commands.Bot):
		def __init__(self, intents: discord.Intents = discord.Intents.all()):
		super().__init__(command_prefix="b!", intents=intents)
		
		# overwrite event handler in bot class itself
		async def on_message(self, message: discord.Message, /) -> None:
		print(f"{message.author} said {message.content}")
		self.process_commands(message)  # otherwise all commands won't trigger
		
		@bot.event  # use the decorator instead of overwriting it in the class directly
		async def on_typing(channel: discord.abc.Messageable, user: discord.Member | discord.User, when):
		await channel.send(f"Naaa, {user.mention} was schreiben wird denn schönes?")
		
		bot = MyBot()
		def start_bot():
		bot.run(os.getenv("TOKEN"))
	\end{lstlisting}
\end{frame}

\begin{frame}{Weitere Kategorie von Events: Klassische Commands}
	Eine spezifischere Teilkategorie von Events sind \tb{Commands}\footnote{Ein el eines Commands findet sich auf dem Beispiel zu \texttt{Cogs}, bei Commands außerhalb von Cogs muss \texttt{@bot.command()} als Decorator genutzt werden!}, mit welchen\newline
	Nutzer:innen gezielt mit einem Bot interagieren.
	
	\medskip
	\pause
	
	Hierbei existiert der \tb{klassische} Message-Command\furl{https://discordpy.readthedocs.io/en/stable/ext/commands/api.html\#discord.ext.commands.command} mit der \tb{Syntax} \pp{prefix|command}.
	\newline
	Also simple \tb{Nachrichten}\footnote{
		Da die Commands auf Nachrichten basieren, muss beim Überschreiben der \texttt{on\_message()}-Funktion ein Aufruf von \texttt{process\_commands(message)} enthalten sein.
	}, welche einer besonderen Syntax folgen.
	
	\medskip
	\pause
	
	Die Interaktion ist ähnlich einer mit \tb{Command-Line} ohne Auto-Completion.
	
	\medskip
	\pause
	
	Die Typen der \tb{Argumente} müssen komplett vom \tb{Developer validiert} werden.
	\newline
	Dabei müssen Fehlerrückmeldungen und Hinweise selbst implementiert werden.
	
	\medskip
	
	Jeder Bot sollte für eine Command-Übersicht einen \tb{Help-Command} bieten.
	
	\medskip
	\pause
	
	Jede Command-Funktion bekommt als \tb{erstes Argument} \pp{commands.Context} und danach die gegebenen Argumente.
	
\end{frame}

\subsection{Die neue Art der direkten Interaktion mit Bots}


\begin{frame}{Application Commands}
	Die neuere Art der Interaktion sind die \tb{Application-Commands}\furl{
		https://discordpy.readthedocs.io/en/latest/interactions/api.html?highlight=app_commands\#decorators
	}\footnote{
		Aufgrund des einheitlichen \texttt{/}-Prefixes oft einfach Slash-Commands genannt.
	}.
	
	\medskip
	\pause
	
	Sie sind direkt in Discord eingebaut und bieten \tb{Vorteile} gegenüber klassischen Commands:
	
	\medskip
	\pause
	
	\begin{itemize}
		\item Native \tb{Übersicht} aller verfügbaren Commands direkt in Discords UI.
		
		\medskip
		\pause
		
		\item \tb{Kommunikation} und \tb{Validierung} der verfügbaren Parameter inklusive Typ.
		
		\medskip
		\pause
		
		\item Ausführung und Antwort kann für andere Nutzer:innen \tb{unsichtbar} sein. 
		
		\medskip
		\pause
		
		\item In Discord eingebautes Rechte-System für die Nutzung von Commands.
	\end{itemize}
	
	\medskip
	\pause
	
	Aus der Developer-Community folgte einiges an \tb{Kritik}, da Slash-Commands die Einführung des \pp{Message Content Intent} ermöglichten und damit die allgemeine Freiheit der Bots einschränken. Discord begründet dies vor allem mit Datenschutz.
	
\end{frame}

\begin{frame}{Application Commands erstellen}
	Die Command-Funktion muss als \tb{ersten Parameter} eine \pp{Interaction}\furl{https://discordpy.readthedocs.io/en/stable/interactions/api.html?highlight=response\%20send\_message\#discord.InteractionResponse} entgegennehmen und kann danach \tb{weitere} (benötigte) Parameter definieren.
	
	\medskip
	\pause
	
	Am \tb{Ende} muss die Funktion über \pp{interaction.response.send_message} auf diese \tb{Antworten}.
	
	\medskip
	\pause
	
	Discord \tb{erwartet} eine sehr schnelle Antwort des Bots. \newline
	Braucht der Command einige API-Calls, wird dieses \tb{Zeitlimit} schnell überschritten.\nsl{es reichen paar api calls}
	\newline
	Bei längeren Operationen ist es daher ratsam, die Interaction mit \pp{.defer()} zu\newline
	\tb{acknowledgen} und nach erfolgreichem Abschluss ein \tb{Followup} zu senden.
	\newline
	Dies geht über \pp{interaction.followup.send()}\furl{https://discordpy.readthedocs.io/en/stable/interactions/api.html?highlight=followup\#discord.Interaction.followup}.
	
\end{frame}

\begin{frame}[fragile]{Slash-Commands}

		from typing import Literal, Optional
		import discord
		from discord import app_commands
		
		# use @app_commands.command instead if command is in Cog (fn must then also take self!)
		@bot.tree.command(name="eaeeae__r_pl_c_m_ntc_s_", description="Convert to replacement case")
		async def replacement_case_command(
		interaction: discord.Interaction,
		text: str,
		mode: Literal["silent", "loud"] | None,

		replacement_text = EAEEAE__r_PL_C_M_NTc_S_(text, eAE___SC_P_=True)
		
		if get_escaped:
		replacement_text = f"```{replacement_text}```"
		
		await interaction.response.send_message(replacement_text, ephemeral=mode == "silent")
	\end{lstlisting}

\begin{frame}{Discord.py: Buttons \& Dropdown}
	Discord bietet Bots die Option \tb{Buttons}\furl{https://discordpy.readthedocs.io/en/latest/interactions/api.html\#id1}\furl{https://github.com/Rapptz/discord.py/blob/master/examples/views/dynamic\_counter.py} und \tb{Select}-Menus\furl{https://discordpy.readthedocs.io/en/latest/interactions/api.html?\#select-menus}\furl{https://github.com/Rapptz/discord.py/blob/master/examples/views/dropdown.py} zu nutzen.
	\newline
	Diese können einer Nachricht angehängt werden.
	
	\medskip
	\pause
	
	Die Elemente verhalten sich \tb{analog} zu klassischen UI-Elementen:
	\pause
	\begin{itemize}
		\item Jedes \tb{erbt} von \pp{discord.ui.View}.
		\item Jedes Element \tb{implementiert} eine \pp{callback}-Funktion, welche bei Press oder Auswahl ausgelöst wird.
	\end{itemize}
	
	\medskip
	\pause
	
	Pro versandtem View existiert \tb{eine} Instanz des Buttons bzw. DropDowns. Dieses kann u.A. weiteren \tb{individuellen} Kontext halten und weitere Methoden definieren.
	
	\medskip
	\pause
	
	Wird der Bot \tb{neu gestartet}, sind alle vorherigen UIs nicht mehr funktionsfähig.
	
\end{frame}


\begin{frame}[fragile]{Tasks}
	
	Alle bisherigen Funktionalitäten erfordern eine explizite Interaktion oder ein bestimmtes Event.
	
	\medskip
	\pause
	
	Für \tb{regelmäßig} auszuführende Aufgaben bietet Discord.py \tb{Tasks}\footnote{Diese liegen in \texttt{discord.ext.tasks}: \furl{https://discordpy.readthedocs.io/en/stable/ext/tasks/index.html}}.
	
	\medskip
	\pause
	
	Tasks werden \tb{gescheduled} und laufen dann in festen \tb{Zyklen} ab.
	
	\medskip
	\pause
	
	Tasks laufen unabhängig eines \pp{Guild}- / \pp{Message}-Kontextes.
	
	\medskip
	
	Daher müssen Informationen wie \pp{Guild}, etc. erst aus dem \pp{Bot} geholt werden\furl{https://discordpy.readthedocs.io/en/stable/ext/commands/api.html\#discord.ext.commands.Bot.get_guild}.
	
	\pause
	
	\begin{lstlisting}[language=pytiny]
		@tasks.loop(seconds=60)  # task runs every 60 seconds
		async def my_background_task(self):      # task is defined in the bot itself, so self is the bot
		channel = self.get_channel(1234567)  # channel ID goes here
		self.counter += 1
		await channel.send(self.counter)
	\end{lstlisting}
	
\end{frame}


\begin{frame}[fragile]{Embeds}
	Discord bietet Bots die Möglichkeit, \tb{Nachrichten} als \pp{Embed}\furl{https://discordpy.readthedocs.io/en/stable/api.html\#embed} \tb{formatiert} zu versenden.
	
	\medskip
	\pause
	
	Diese bieten eine Reihe Formatierungs- und Strukturierungsoptionen.
	
	\medskip
	\pause
	
	Embeds werden einer Nachricht z.B. bei \pp{.send()} \tb{angehängt}.
	\newline
	Sie dürfen mehr Text als normale Nachrichten enthalten.
	
	\medskip
	\pause
	
	Die \tb{Limits} für Embeds können hier\furl{https://discord.com/developers/docs/resources/channel\#embed-object-embed-limits} eingesehen werden.
	
	
\end{frame}

\begin{frame}{Hinweise \& Best Practices}
	\begin{itemize}
		\item Hinzufügen/ Entfernen von z.B. Rollen: \pp{set}-Operationen!
		
		\pause
		
		\item Nutze vernünftiges \tb{Logging}, da das Programm viel im Hintergrund laufen wird!
		
		\pause
